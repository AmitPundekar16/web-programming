Objective: To perform asynchronous form submission using AJAX to update web content without reloading the page.

Section 1: Understanding AJAX

What is AJAX?

AJAX (Asynchronous JavaScript and XML) is a technique that allows web pages to send and receive data from a server without reloading the page. Despite the name, modern AJAX almost always uses JSON instead of XML.

In a traditional form submission, clicking "Submit" sends the form data to the server and the browser reloads the entire page with the server's response. With AJAX, JavaScript intercepts the form submission, sends the data in the background using fetch(), and updates only the relevant part of the page.

Key difference: Traditional forms use the browser's default behaviour (action + method attributes cause a full page reload). AJAX uses JavaScript to send the request and handle the response — the page never reloads.

The AJAX Form Submission Pattern

// 1. Prevent the default form submission (which would reload the page)
form.addEventListener('submit', async function(event) {
    event.preventDefault();

    // 2. Collect form data
    const title = document.getElementById('title').value;

    // 3. Send data to the server using fetch() with POST method
    const response = await fetch('https://api.example.com/posts', {
        method: 'POST',                              // HTTP method
        headers: { 'Content-Type': 'application/json' },  // tell server we're sending JSON
        body: JSON.stringify({ title: title })        // convert JS object to JSON string
    });

    // 4. Parse the server's response
    const result = await response.json();

    // 5. Update the page with the result (no reload!)
    document.getElementById('output').innerHTML = `<p>Created: ${result.title}</p>`;
});

Summary of Concepts

Concept What You Learned
AJAX Send and receive data from a server without reloading the page. The core technique behind modern interactive web apps.
event.preventDefault() Stops the browser's default form submission (which causes a page reload). Required for AJAX form handling.
POST method Creates a new resource on the server. Pass method: 'POST' in the fetch options.
PUT method Replaces an existing resource. Pass method: 'PUT' with the full updated object in the body.
DELETE method Removes a resource from the server. Pass method: 'DELETE' — typically no body is needed.
JSON.stringify() Converts a JavaScript object into a JSON string for sending in a request body.
Content-Type header Tells the server what format the data is in. 'application/json' means the body is JSON.
response.ok Boolean that is true if the HTTP status is 200–299. Use it to check if the request succeeded.
DOM manipulation after AJAX After a successful request, update only the affected part of the page using innerHTML or .remove().
